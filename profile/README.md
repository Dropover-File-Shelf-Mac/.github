## What is Dropover?

**Dropover** adds a temporary shelf to your Mac. Drag files or folders onto it, switch to another window or app, then drag from the shelf to the final destination. If you release the drag by mistake, files stay on the shelf instead of disappearing. Supports **macOS High Sierra through Sonoma**, **Intel and Apple Silicon**.

---

## Compatibility

Dropover works with any app that accepts standard macOS drag and drop: Finder, browsers, email clients, design tools, IDEs, and most file-based apps. You drag from Finder or from an app’s document list onto the shelf, then from the shelf to the destination. The shelf stays on top so you can switch windows or Spaces and then drop. Some web-based upload areas also accept drops from the shelf as long as the OS allows it. If an app uses a custom drag implementation, behavior may vary. No installation inside other apps is required; Dropover is system-wide.

## How to Trigger the Shelf & Gestures

**Opening the shelf:**
- **Shake** the mouse or trackpad (configurable in Preferences).
- **Menu bar icon** — Click the Dropover icon and choose to show the shelf.
- **Keyboard shortcut** — Set a global shortcut (e.g. Option + D) in Preferences to toggle the shelf.
- **Edge or corner** — Some setups allow triggering by moving the cursor to a screen edge (if enabled).

**While dragging:**
- Drag files over the **trigger zone** or **bubble**; the shelf appears and you drop onto it.
- Drag from the shelf to any app or Finder window to **move** or **copy** (depending on Dropover settings and modifier keys).

**Shelf behavior:**
- The shelf can be set to stay on top of other windows so it’s always visible when needed.
- You can stack multiple items from different folders and drop them together into one destination.

Check **Dropover → Preferences** (or **Settings**) for trigger sensitivity, shelf position, and shortcut.

## Copy vs Move & Shelf Settings

**Copy vs Move:** In Preferences you can set the default when dropping from the shelf (e.g. copy into destination). Using **Option** (or another modifier) while dropping often toggles between copy and move—see the app’s help or Preferences for the exact key.

**Shelf settings:**
- **Position** — Corner or side of the screen so it doesn’t block work.
- **Appearance** — Size, opacity, and style of the shelf and bubble.
- **Trigger sensitivity** — How strong the shake or how long the hover to open the shelf.
- **Menu bar** — Show or hide the icon; some users keep it for quick access without Dock.

Tuning these makes Dropover fit different workflows (e.g. lots of uploads vs. occasional file moves).

## Use Cases

- Moving or copying files between several folders or drives without many Finder windows
- Collecting files from different places before uploading to a web app or email
- Organizing downloads or project files by grabbing from multiple locations and dropping into one folder
- Dragging assets into design or dev tools from various sources
- Keeping a set of files "in hand" while deciding where they go

---

## Tips & Best Practices

- Set a **keyboard shortcut** you'll remember for showing the shelf.
- **Stack** several items on the shelf, then drop them all into one folder or upload dialog.
- Use **Split View** or **Spaces** with Dropover to move files between full-screen apps.
- Keep the **Menu Bar** icon enabled for access without using Dock space.
